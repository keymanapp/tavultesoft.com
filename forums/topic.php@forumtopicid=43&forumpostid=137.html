Hi Paul,

One question? Can the language group you are working with be fully represented in Unicode? Ie. are there and missing characters? If not then it should be straight forward.

SIL have some useful resources:
    Converting Keyman 3.2 Keyboards to Keyman 6: How to get your legacy keyboards to work where they shouldnâ€™t. http://scripts.sil.org/cms/scripts/page.php?site_id=nrsi&item_id=Keyman32to60Tutorial&highlight=keyman~3.2

    How to Write a Conversion Mapping for your Legacy Font.
    http://scripts.sil.org/cms/scripts/page.php?site_id=nrsi&item_id=UTTWriteMap

    Unicode Transition Tutorial Links
    http://scripts.sil.org/cms/scripts/page.php?site_id=nrsi&cat_id=UnicodeTutorials

SIL also produce a encoding converter called TECKit http://scripts.sil.org/cms/scripts/page.php?site_id=nrsi&cat_id=TECkit. You can create a mapping file for your legacy fonts to unicode. There is a plugin for Microsoft Word that allows you to run the conversion in place within the word document. Its also possible to daisy chain conversions. It can handle unicode normalization and allows you to change fonts during the conversion process.

Should enable you to move your old word files to unicode.
