I support many language teams whose computers range from XP through Win 7. So their Keyman versions range from 6.2 to 8. If I update my keyboards with Developer 8, will Keyman 6.2 still be able to use them?
# RE: Backwards Compatibility   2011-11-06 15:33:11.233
Marc Durdin
Tavultesoft Staff
Yes, it is possible to compile Keyman 6 keyboards with Keyman Developer 8. However, you will of course be unable to use Keyman 8 features such as keyboard options, or the visual editing tools. You may find that if you open the keyboard and use the visual editing tools as opposed to the source editor, that Keyman Developer will update the file version to 8.0, in which case you'll need to change the file version in the source tab back to '6.0'.

I'm not sure if it is possible to create Keyman 6.0 compatible packages with Keyman Developer 8.0 -- it's not a scenario I tested; you'd have to try it and see!
