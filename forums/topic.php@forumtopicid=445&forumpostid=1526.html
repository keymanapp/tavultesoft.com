Hi,

I am trying to work in Excel 2013 with one of my keyboards.
I have the following rules
:

store(vowelkeys)      [K_O]     [SHIFT K_O]     [SHIFT K_E]     [K_E]     [SHIFT K_U]     [K_U]     [SHIFT K_I]     [K_I]     [SHIFT K_A]     [K_A]
store (vowelunicode)     'ඔ'    'ඕ'    'ඒ'    'එ'    'ඌ'    'උ'    'ඊ'    'ඉ'    'ආ'    'අ'
group(main) using keys
+ any(vowelkeys)> index(vowelunicode,1)
'අ'+[K_A] > 'ආ'


the idea is that one press of 'a' will produce 'අ' and two consecutive presses 'ආ'. In Excel 2013, this works fine unless I do it in the beginning of an empty cell. for example if I select an empty cell and type 'aa' it will produce 'අඅ'. However if I delete them and retype 'aa' it will produce 'ආ' as intended.

Marc Durdin
Tavultesoft Staff
Yes, this is a known limitation of using Keyman in Excel. The problem is that the first letter is actually typed into a different place than subsequent letters, so it does not appear to Keyman that it is part of the same sequence. Workarounds include pressing F2 to start editing the cell, or pressing e.g. space,backspace.

It's not great, but there's not a lot we can do at present to improve this.
