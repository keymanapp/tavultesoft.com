# Ctrl+Backspace not working properly   2006-10-04 01:42:15.677
Alex Bepple
In quite a few applications, Ctrl+Backspace deletes the word in front of the cursor. However, when a Keyman keyboard is active, this does not work. Firefox does nothing at all, but quite a few applications print a rectangle, e.g. Microsoft Word, Textpad, UEStudio. According to UEStudio the character has the hex value 7F. Is there any workaround or setting to correct this behavior?

Hi Alex,

This issue is due to a bug in Keyman, but unfortunately I can't find any way to correct it. To work around it (with the same result), the fastest method would probably be to use Ctrl+Shift+Left arrow and then press Delete. This should work in most applications.

We'll make sure to correct this issue will be corrected in Keyman 7, which is currently in beta testing.
